# HYGGE HP｜イベント・ブログのGAS連携 セットアップ手順

スプレッドシートに入力するだけで、HPの「イベント・お知らせ」と「ブログ」が更新できるようになります。
作業は最初の1回だけ。以降はスプレッドシートを編集するだけで反映されます。

---

## 1. スプレッドシートを用意する

Googleスプレッドシートを新規作成し、**シートを2つ**用意します（下のタブ名を正確に）。

### シート①：`イベント`
1行目に見出し、2行目からデータを入れます。

| A 日付 | B タイトル | C 詳細 | D 画像 | E タグ | F 表示 |
|--------|-----------|--------|--------|--------|--------|
| 2026.07.09 | HYGGE グランドオープン | 植物と暮らし雑貨の新しいお店がオープン。 | （画像リンク） | NEW OPEN | |
| 2026.07.09 | Spice Curry momo×Hygge Brew And | 駐車場でスパイスカレーとコーヒーの出店。 | （画像リンク） | コラボ | |

### シート②：`ブログ`

| A 日付 | B タイトル | C 抜粋 | D 画像 | E 本文 | F スラッグ | G リンク | H 表示 |
|--------|-----------|--------|--------|--------|-----------|----------|--------|
| 2026.07.10 | ビカクシダの育て方 | 水やりと置き場所の基本を解説。 | （画像リンク） | 板付けビカクシダは…（記事本文。改行で段落） | bikakushida-care | | |
| 2026.07.03 | 観葉植物のある暮らし | お部屋に緑を取り入れるコツ。 | （画像リンク） | まずは育てやすい一鉢から…（本文） | green-life | | |

> **各列の意味**
> - **画像**：画像の直リンクURL、または **Googleドライブの共有リンク** でOK（自動で表示用に変換されます）
> - **本文**：記事の中身。**空の行で段落が分かれます**。詳細ページに表示されます
> - **スラッグ**：記事URLに使う半角英数（例 `bikakushida-care` → `/blog/bikakushida-care.html`）。空なら自動で `post-行番号`
> - **タグ**（イベントのみ）：「NEW OPEN」「コラボ」などの小ラベル（任意）
> - **リンク**：通常は空でOK。**外部記事に飛ばしたい時だけ**URLを入れる（入れるとそのカードは詳細ページではなく外部URLへ）
> - **表示**：`FALSE` と入れると非表示。空 or `TRUE` で表示

---

## 2. 画像の入れ方（おすすめ：Googleドライブ）

1. Googleドライブに画像をアップロード
2. 画像を右クリック →「共有」→「リンクを知っている全員」に変更
3. 「リンクをコピー」して、スプレッドシートの **D列（画像）** に貼り付け

これだけでOK。直リンクURL（`https://〜.jpg` など）を持っている場合はそのまま貼ってもOKです。

---

## 3. GASを設定する（最初の1回だけ）

1. スプレッドシート上部メニュー → **拡張機能 → Apps Script** を開く
2. 既存のコードを全部消して、**`Code.gs` の中身を貼り付け**て保存
3. 右上 **デプロイ → 新しいデプロイ**
4. 種類は **ウェブアプリ** を選び、下記の通り設定：

   | 項目 | 設定値 |
   |------|--------|
   | 次のユーザーとして実行 | **自分** |
   | アクセスできるユーザー | **全員** |

5. **デプロイ** を押す → 表示される **ウェブアプリのURL**（末尾が `/exec`）をコピー

---

## 4. HPにURLを貼る（最初の1回だけ）

`index.html` を開き、下の行を探します（ページ下のほうの `<script>` 内）：

```js
var GAS_URL = '★ここにGASウェブアプリのURLを貼る（末尾は /exec）★';
```

`★…★` の部分を、手順3でコピーした **/exec のURL** に置き換えて保存します。

```js
var GAS_URL = 'https://script.google.com/macros/s/AKfyc.../exec';
```

保存してHPを再読み込みすると、スプレッドシートの内容が表示されます。

---

## 5. 以降の更新（毎回の運用）

- イベントやブログを追加 → スプレッドシートに **行を追加**
- 内容を直す → セルを編集
- 一時的に隠す → **F列に `FALSE`**
- 消す → その行ごと削除

保存するだけでHPに反映されます（再デプロイは不要）。

---

## 注意点

- **再デプロイでURLが変わった場合**は、`index.html` の `GAS_URL` も貼り直してください
  （「デプロイの管理」から既存デプロイを**編集**すればURLは変わりません。基本はこちら推奨）
- `アクセスできるユーザー：全員` にしないとHPから読み込めません
- URL未設定の間は、`index.html` に書かれている初期カードがそのまま表示されます（壊れません）

---

## 困ったとき

- 反映されない → URLの末尾が `/exec` か、`アクセス：全員` か確認
- 画像が出ない → ドライブの共有が「リンクを知っている全員」になっているか確認
- それでもダメなら、Claudeに「GAS連携が反映されない」と相談してください

# ブログの詳細ページ（SEO対応・静的生成）

イベント・お知らせは「シート保存だけで即反映」ですが、**ブログ記事の詳細ページは検索エンジン対策（SEO）のため、別の仕組み**で動きます。

## 仕組み

```
スプレッドシート（本文・スラッグを記入）
  ↓ build-blog.js が読み取り
記事ごとのHTMLを自動生成（タイトル・説明・OGP・本文入り）
  ↓ GitHub に push
Vercel が公開（/blog/スラッグ.html）
```

- トップページのブログ一覧（カード）は今まで通りシートから自動表示され、押すと生成された詳細ページ `/blog/スラッグ.html` に飛びます。
- 詳細ページは記事ごとに固有のURL・タイトル・説明・OGP画像・構造化データを持つので、検索やSNSシェアに強くなります。
- `sitemap.xml` も自動生成されます。

## 最初の1回だけの設定

1. `build-blog.js` を開き、先頭の **`SITE_URL`** を本番の公開URL（例 `https://hygge-kumamoto.com`）に変更
2. GitHubリポジトリに `build-blog.js` と `.github/workflows/build-blog.yml` を置く（Coworkで反映済み）
3. これで毎日自動でブログが再生成・公開されます

## 記事の公開フロー（毎回）

1. `ブログ`シートに行を追加し、**本文**と**スラッグ**を記入
2. 保存して待つ → **毎朝6時に自動でビルド・公開**されます
3. すぐ反映したい時は、GitHubの **Actions → 「Build Blog」→ Run workflow**（手動実行）を押す

> ※トップの一覧（カード）はシート保存ですぐ反映されますが、**詳細ページ本体は自動ビルドのタイミング**で作られます。急ぎは手動実行で即時生成できます。

## 注意

- スラッグは記事ごとに**重複しない半角英数**にしてください（同じだと上書きされます）
- 一度公開したスラッグは**変えない**のが安全（URLが変わると検索評価がリセットされます）
- 本文の段落は**空行**で区切ります
